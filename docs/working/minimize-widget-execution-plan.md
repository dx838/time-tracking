## 一句话定义

- [x] 本方案要做的是：把“最小化”从“直接隐藏到托盘”扩展为“隐藏主窗口并显示吸附在屏幕左右侧的小挂件”
- [x] 本方案不做的是：自由漂浮窗、迷你 Dashboard、桌面玩具、强告警面板

## 当前已确认结论

- [x] `关闭` 继续维持当前托盘/后台运行语义
- [x] `最小化` 单独新增“挂件”行为，不与 `关闭` 混用
- [x] 最小化设置采用开关表达：关闭为任务栏，开启为挂件
- [x] 挂件采用独立窗口，不把主窗口硬改成迷你版
- [x] 挂件只允许吸附左侧或右侧
- [x] 挂件允许拖动，但仅为“吸边可拖”
- [x] 挂件要记住上次的吸附侧与垂直位置
- [x] 挂件状态文案使用短标签
- [x] 挂件状态灯使用呼吸效果
- [x] 状态灯颜色采用绿色、黄色、灰色、红色
- [x] 状态灯呼吸仅作用于小状态灯，不扩散到整块挂件

## 已定稿细节

- [x] 展开方式最终定为：点击展开 + 自动半收起
- [x] 不允许悬停展开
- [x] 半收起宽度最终定为 `16px`
- [x] 展开宽度最终定为 `112px`
- [x] 首版不展示当前应用图标
- [x] “打开主界面”采用文字按钮
- [x] 首版不加入“全屏应用时自动隐藏”

## 产品目标

- [x] 让用户在最小化后仍能低打扰地获知追踪状态
- [x] 保持 Quiet Pro 的克制与长期使用舒适度
- [x] 尽量复用现有 tracking 状态流和桌面行为体系
- [x] 控制新增复杂度，不把多窗口行为扩散到无关层

## 明确非目标

- [x] 不做任意位置自由拖拽
- [x] 不做桌面常亮大面板
- [x] 不做迷你图表、迷你历史、迷你统计中心
- [x] 不做挂件主题商店、皮肤系统、颜色自定义
- [x] 不做首版多挂件、多实例或多样式布局

## 交互定义

### 主窗口与挂件关系

- [x] 主窗口与挂件是两个明确职责分离的窗口
- [x] 主窗口负责完整应用体验
- [x] 挂件只负责最小化后的轻量状态感知与快捷操作
- [x] 任一时刻不应让主窗口和挂件都同时占据主要注意力

### 最小化行为

- [x] 当 `minimize_behavior = widget` 时，点击主窗口最小化按钮
- [x] 主窗口不进入任务栏最小化可见态，而是隐藏
- [x] 挂件窗口显示
- [x] 挂件窗口恢复到上次记忆位置
- [x] 若没有历史位置，则使用安全默认位置

### 关闭行为

- [x] 当 `close_behavior = tray` 时，关闭主窗口仍进入当前托盘语义
- [x] 当 `close_behavior = exit` 时，关闭主窗口仍退出应用
- [x] “挂件模式”不改变关闭行为定义

### 恢复主界面行为

- [x] 点击挂件中的“打开主界面”后，主窗口显示
- [x] 主窗口显示时自动获得焦点
- [x] 主窗口恢复时挂件隐藏
- [x] 托盘菜单“打开主界面”仍直接恢复主窗口

### 半收起/展开行为

- [x] 挂件默认处于半收起状态
- [x] 半收起状态只露出窄边和状态灯
- [x] 展开后显示全部首版信息
- [x] 展开与收起都保持低动效、短动效
- [x] 不允许在展开时跳出明显的桌面干扰感

### 拖拽行为

- [x] 挂件提供明确拖拽热区
- [x] 按住拖拽热区时可以上下移动
- [x] 拖拽穿过屏幕中线可切换左右吸附边
- [x] 松手后自动吸附最近边
- [x] 吸附后重新计算半收起和展开位置
- [x] 拖拽完成后立即写入存储
